# Leone1111_online_notes 项目规则

## 项目简介

Leone1111_online_notes 是一个网页端系统，面向想学习的用户，用来上传和管理读书笔记、科研笔记、代码笔记和项目笔记。

## 当前产品目标

本项目是个人科研与知识分享网站，核心场景包括：

1. 科研笔记：记录文献阅读、科研想法、实验记录。
2. 代码项目：展示和下载个人小程序、项目压缩包。
3. 读书笔记：展示书籍阅读笔记和总结。

## 用户权限设计

- 游客：可以浏览公开内容、查看文章、下载公开文件。
- 网站拥有者：可以进入管理模式，上传文件、发布笔记、编辑内容、删除内容。
- 网站暂不开放普通用户注册和评论功能。

## 文件上传需求

需要支持以下文件类型：

- PDF
- Word 文档
- Markdown
- 图片
- 项目压缩包，例如 zip

上传功能应优先保证安全性，不要把管理员密钥暴露在前端代码中。

## 回答要求

- 每次回答都要用用户能听懂的话解释。
- 避免只说专业术语；如果必须使用专业词，要顺带解释它是什么意思。
- 先说结论，再说明原因和下一步。
- 不确定的事情要明确说明，不要装作确定。

## 文件操作规则

- 不要乱删文件。
- 删除、覆盖、移动重要文件之前，必须先说明原因并征得用户同意。
- 修改文件前要先确认改动范围，尽量只改和当前任务有关的文件。
- 如果发现已有改动，不要随便回退，要先判断这些改动是不是用户留下的。

## 开发协作规则

- 优先保持项目现有结构和写法一致。
- 新增功能时，尽量让用户知道改了哪里、为什么这么改。
- 完成任务后，要简单说明实际完成的内容。
